# Исправление штрафа 5 ₽ вместо лишения прав

## Причина бага

При обновлении контента скрипт `web/scripts/audit-pdd-content.py` в функции `parse_numbers_from_sanction` брал **первый числовой диапазон** в санкции и считал середину:

```
Текст: «Штраф 7500 руб. или лишение прав на 4-6 месяцев»
NUM_RANGE → находит «4-6» → (4+6)//2 = 5 → fine = 5  ❌
```

Проверка `if "лишение" in low and "руб" not in low` не срабатывала, потому что в строке **есть и «руб», и «лишение»**.

В игре поле `fine: -1` — конвенция для лишения прав; UI (`ErrorInspectorPanel`) показывает «Лишение прав на срок до N мес.» только при `fine === -1` и заполненном `licenseRevokeMonths`.

**Исправление парсера:** если в санкции есть «лишение» — сразу возвращать `-1`; диапазоны с «мес» / «месяц» после чисел игнорировать.

Тексты ошибок (`C_ERRORINFO` / `error_info`) **не менялись** — исправлено только игровое поле `fine` и пересобран `missions.json`.

## Верификация

После `npm run build:data` в `missions.json` для всех 5 веток:

```json
"fine": -1,
"licenseRevokeMonths": 6
```

Spot-check в игре: сцена **3.7**, ответ 3 (кольцо) → popup **«Лишение прав на срок до 6 мес.»**, не «Штраф: 5 ₽».
